The Router can be configured with multiple options depending on the situation. These options are passed as second object to config function and this is an optional parameter.
Custom Error Page URL
By setting this option we can setup cutom template url for our error pages. If not set, the router will use a default error message.
$Router.config([
    ...
], {
    customErrorPageUrl: 'partial/customError.html'
});
Custom Error Page Template
By setting this option we can setup cutom template for our error pages. If not set, the router will use a default error message.
$Router.config([
    ...
], {
    customErrorPageTemplate: `<div>This is a custom Error Template</div>`
});
Show/ Hide Error Pages
This option is used to show/ hide error pages when the router cannot find configured partials. If not set, the router will show a default error message.
This option defaults to true
$Router.config([
    ...
], {
    showErrorPage: true
});
Activate Links
When this option is set as 'true' it will set an active marker on the links that are currently clicked, by adding an additional class to the anchor tag. By default the class added is active.
This option defaults to true
$Router.config([
    ...
], {
    activateLinks: true
});
Activate Link Class
This option is used to override the additional class added to the anchor tag that is active.
This option defaults to active
$Router.config([
    ...
], {
    activateLinkClass: 'active'
});
Default Link Class
This option is used when there are additional classes that should be added to the anchor tags other than activateLinkClass.
This option defaults to empty.
$Router.config([
    ...
], {
    defaultLinkClass: 'someOtherClass'
});
Before Route Change
This is a life-cycle hook in Router. Pass a function to this option, to be called each time before the route is changed. The function will be provided with the active path as a parameter
This option defaults to undefined
$Router.config([
    ...
], {
    beforeRouteChange: (activePath) => {console.log('Before Route Change =>',activePath);}
});
After Route Change
This is a life-cycle hook in Router. Pass a function to this option, to be called each time after the route has changed. The function will be provided with the active path as a parameter
This option defaults to undefined
$Router.config([
    ...
], {
    afterRouteChange: (activePath) => {console.log('After Route Change =>',activePath);}
});
On Route Change Error
This is a life-cycle hook in Router. Pass a function to this option, to be called every time there is a error during route change. The function will be provided with the active path as a parameter
This option defaults to undefined
$Router.config([
    ...
], {
    onRouteChangeError: (activePath) => {console.log('Route Change Error =>',activePath);}
});
Router Outlet Selector
This option can be over-rided if we need a custom selector to act as our Router outlet.
This option defaults to router-outlet
$Router.config([
    ...
], {
    routerOutletSelector: 'router-outlet'
});
Routing Strategy
This option determines your routing strategy. This will by default set the strategy as HTML5 compliant History API. But if you want to use legacy hashing strategy change the option to true
This option defaults to false
$Router.config([
    ...
], {
    useHashStrategy: false
});
